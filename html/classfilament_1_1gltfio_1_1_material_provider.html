Detailed Description

Interface to a provider of glTF materials (has two implementations).

  • The JitShaderProvider implementation generates materials at run time (which can be slow) and requires the filamat library, but produces streamlined shaders. See createJitShaderProvider().
  • The UbershaderProvider implementation uses a small number of pre-built materials with complex fragment shaders, but does not require any run time work or usage of filamat. See createUbershaderProvider().

Both implementations of MaterialProvider maintain a small cache of materials which must be explicitly freed using destroyMaterials(). These materials are not freed automatically when the MaterialProvider is destroyed, which allows clients to take ownership if desired.
